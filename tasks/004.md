# Task 004: 实验性本地 VAD 开关

## 目标

在设置中添加"实验性功能"开关，允许用户启用本地 VAD（Voice Activity Detection，语音活动检测），而不是完全依赖 ElevenLabs 的云端 VAD。

## 背景

### 当前实现（Task 001）
- **录音方式**：用户手动点击录音按钮开始/停止
- **ASR 服务**：ElevenLabs Scribe
- **VAD**：无（用户手动控制）

### 为什么需要本地 VAD？

1. **自动检测语音结束**：用户说完话后自动停止录音，无需手动点击
2. **更自然的交互**：类似 Siri/Google Assistant 的体验
3. **降低延迟**：本地 VAD 可以更快响应
4. **离线可用**：不依赖网络（VAD 部分）
5. **节省成本**：减少无效音频上传到云端

### 开关状态说明

**关闭（默认）**：
- 手动控制录音开始/停止
- 用户点击录音按钮停止

**开启（实验性）**：
- 点击录音按钮开始
- 检测到静音后自动停止
- 提供视觉反馈（波形图、静音倒计时等）

## 技术方案

### 1. 本地 VAD 库选择

#### 方案 A：@ricky0123/vad-react-native（推荐）

**优点**：
- 基于 Silero VAD（业界标准）
- 专为 React Native 优化
- 轻量级（<5MB）
- 实时处理
- 离线运行

**缺点**：
- 需要额外依赖

**安装**：
```bash
npm install @ricky0123/vad-react-native
```

**使用示例**：
```typescript
import { useMicVAD } from '@ricky0123/vad-react-native';

const vad = useMicVAD({
  onSpeechStart: () => console.log('Speech started'),
  onSpeechEnd: (audio) => console.log('Speech ended'),
  onVADMisfire: () => console.log('False positive'),
  positiveSpeechThreshold: 0.7, // 语音检测阈值
  minSpeechFrames: 4, // 最少语音帧数
  preSpeechPadFrames: 1, // 语音开始前的缓冲帧
  redemptionFrames: 8, // 静音容忍帧数（防止误判）
});

vad.start(); // 开始监听
vad.pause(); // 暂停
```

#### 方案 B：react-native-audio-api + 自定义 VAD

**优点**：
- 项目已有依赖（`react-native-audio-api`）
- 完全可控

**缺点**：
- 需要自己实现 VAD 算法
- 性能和准确度需要调优

**实现思路**：
```typescript
// 简单的能量阈值 VAD
function detectSpeech(audioBuffer: Float32Array): boolean {
    const rms = Math.sqrt(
        audioBuffer.reduce((sum, val) => sum + val * val, 0) / audioBuffer.length
    );
    const threshold = 0.01; // 需要调整
    return rms > threshold;
}
```

#### 方案 C：Web Audio API + 自定义 VAD（仅 Web）

**优点**：
- 浏览器原生支持
- 无额外依赖

**缺点**：
- 仅支持 Web
- 不适合跨平台 App

### 推荐：方案 A（@ricky0123/vad-react-native）

理由：
- 成熟可靠（基于 Silero VAD）
- 跨平台支持
- 易于集成
- 性能优秀

## 代码结构

### 1. 创建 VAD Hook

```typescript
// sources/hooks/useLocalVAD.ts
import { useEffect, useRef, useState } from 'react';
import { useMicVAD } from '@ricky0123/vad-react-native';
import { DEFAULT_VAD_CONFIG, VADConfig } from '@/services/vadConfig';

export interface UseLocalVADProps {
    enabled: boolean;
    config?: Partial<VADConfig>;
    onSpeechStart?: () => void;
    onSpeechEnd?: (audio: Float32Array) => void;
    onVADMisfire?: () => void;
}

export function useLocalVAD(props: UseLocalVADProps) {
    const [isListening, setIsListening] = useState(false);
    const [isSpeaking, setIsSpeaking] = useState(false);

    const config = { ...DEFAULT_VAD_CONFIG, ...props.config };

    const vad = useMicVAD({
        // 启用/禁用
        startOnLoad: false,

        // Silero VAD 参数
        positiveSpeechThreshold: config.positiveSpeechThreshold,
        negativeSpeechThreshold: config.negativeSpeechThreshold,
        minSpeechFrames: config.minSpeechFrames,
        preSpeechPadFrames: config.preSpeechPadFrames,
        redemptionFrames: config.redemptionFrames,
        frameSamples: config.frameSamples,

        // 回调
        onSpeechStart: () => {
            console.log('[VAD] Speech started');
            setIsSpeaking(true);
            props.onSpeechStart?.();
        },

        onSpeechEnd: (audio) => {
            console.log('[VAD] Speech ended');
            setIsSpeaking(false);
            props.onSpeechEnd?.(audio);
        },

        onVADMisfire: () => {
            console.log('[VAD] Misfire (false positive)');
            props.onVADMisfire?.();
        },
    });

    const start = async () => {
        if (!props.enabled) {
            console.warn('[VAD] VAD is disabled in settings');
            return;
        }
        await vad.start();
        setIsListening(true);
    };

    const pause = async () => {
        await vad.pause();
        setIsListening(false);
        setIsSpeaking(false);
    };

    useEffect(() => {
        return () => {
            // 清理
            vad.pause();
        };
    }, []);

    return {
        start,
        pause,
        isListening,
        isSpeaking,
        loading: vad.loading,
        userSpeaking: vad.userSpeaking,
    };
}
```

// sources/hooks/useVoiceRecording.ts (新增 VAD 支持)
import { useLocalVAD } from './useLocalVAD';
import { useSetting } from '@/sync/storage';

export function useVoiceRecording(): UseVoiceRecordingReturn {
    const [isRecording, setIsRecording] = useState(false);
    const recordingRef = useRef<Audio.Recording | null>(null);

    // 读取 VAD 设置
    const vadEnabled = useSetting('experimentalLocalVAD');

    // 初始化 VAD
    const vad = useLocalVAD({
        enabled: vadEnabled,
        onSpeechStart: () => {
            console.log('Speech detected, recording...');
        },
        onSpeechEnd: async (audio) => {
            console.log('Silence detected, stopping recording...');
            // 自动停止录音
            await stopRecording();
        },
    });

    const startRecording = useCallback(async () => {
        try {
            // ... 现有录音逻辑

            // 如果启用 VAD，开始监听
            if (vadEnabled) {
                await vad.start();
            }

            setIsRecording(true);
        } catch (error) {
            // ...
        }
    }, [vadEnabled, vad]);

    const stopRecording = useCallback(async () => {
        try {
            // 停止 VAD
            if (vadEnabled) {
                await vad.pause();
            }

            // ... 现有停止录音逻辑
        } catch (error) {
            // ...
        }
    }, [vadEnabled, vad]);

    return {
        isRecording,
        isSpeaking: vad.isSpeaking, // 新增：是否正在说话
        startRecording,
        stopRecording,
        cancelRecording,
    };
}

## 性能和电池考虑

### VAD 性能影响

| 指标 | 无 VAD | 启用 VAD |
|------|--------|----------|
| CPU 使用 | ~5% | ~8-12% |
| 内存占用 | ~10MB | ~15MB |
| 电池消耗 | 低 | 中 |
| 延迟 | 0ms | ~50-100ms |

### 优化策略

1. **仅录音时启用**：不录音时暂停 VAD
2. **参数优化**：调整帧大小和阈值
3. **后台限制**：App 进入后台时停止 VAD
4. **电池感知**：低电量模式下禁用

## 边界情况处理

### 1. 环境噪音
- **问题**：持续噪音可能被误判为语音
- **解决**：调整 `positiveSpeechThreshold`
- **UI 提示**：在嘈杂环境建议关闭 VAD

### 2. 说话中间停顿
- **问题**：停顿可能触发提前结束
- **解决**：增加 `redemptionFrames`（容忍帧数）
- **默认值**：8 帧（约 0.5-1 秒）

### 3. VAD 误触发
- **问题**：短暂噪音触发录音开始
- **解决**：使用 `minSpeechFrames` 过滤短暂噪音
- **回调**：`onVADMisfire` 通知用户

### 4. 权限问题
- **问题**：麦克风权限被拒绝
- **解决**：降级到手动模式
- **UI 提示**：提示用户授予权限

### 5. 性能问题
- **问题**：低端设备卡顿
- **解决**：提供"低功耗模式"选项
- **降级**：自动禁用 VAD

## 风险和注意事项

### 风险
1. **误判率**：VAD 可能在嘈杂环境误判
2. **兼容性**：部分设备可能不支持
3. **电池消耗**：持续音频处理耗电
4. **性能影响**：低端设备可能卡顿

### 缓解措施
1. **默认关闭**：作为实验性功能，默认禁用
2. **降级策略**：失败时自动降级到手动模式
3. **用户教育**：清晰的设置说明和警告
4. **性能监控**：监测 CPU 和内存使用
5. **易于禁用**：一键关闭 VAD
