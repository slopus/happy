# Task 005: 改进系统通知信息的具体性

## 目标

将系统popup通知信息从通用提示改为更具体的信息，明确显示：
1. 是哪个任务/会话
2. 具体需要什么（权限请求 vs 新指令）

## 背景

### 当前问题

目前的系统通知可能过于笼统，用户看到通知时不清楚：
- 是哪个AI会话需要注意
- 是需要授权某个权限
- 还是AI在等待用户输入新的指令

这导致用户体验不佳，尤其是当有多个会话同时运行时。

### 期望效果

通知应该包含：
- **会话标识**：会话名称/ID
- **请求类型**：
  - "需要权限：[具体权限名称]"
  - "等待指令"
  - "任务完成"
- **简短描述**：AI正在做什么或需要什么

## 需求分析

### 1. 识别通知触发点

需要找到所有触发系统通知的地方：
- Permission requests（权限请求）
- Waiting for user input（等待用户输入）
- Task completion（任务完成）
- Error states（错误状态）

### 2. 通知信息结构

```typescript
interface NotificationContent {
    // 会话信息
    sessionId: string;
    sessionName?: string;

    // 通知类型
    type: 'permission' | 'input' | 'completion' | 'error';

    // 具体内容
    title: string;        // 主标题
    body: string;         // 详细描述
    action?: string;      // 可选的行动建议

    // 权限相关（如果type是permission）
    permission?: {
        name: string;     // 权限名称
        reason: string;   // 请求原因
    };
}
```

```
内容：网络连接失败，请检查网络设置
```

### 4. 标题设计原则

1. **包含会话名称**：让用户立即知道是哪个任务
   - 格式：`"[会话名称]" [状态描述]`
   - 示例：`"数据分析" 等待指令`

2. **状态清晰**：用简短的动词或状态词
   - 需要权限 / 等待指令 / 已完成 / 遇到错误

3. **长度控制**：
   - 会话名称：最多20个字符（超出用...截断）
   - 总标题：最多40个字符（考虑移动设备通知栏）

4. **优先级顺序**：
   - 高优先级：权限请求、错误
   - 中优先级：等待输入
   - 低优先级：任务完成

### Phase 2: 设计通知内容生成器

创建 `sources/services/notificationContent.ts`:

```typescript
import { Session } from '@/sync/types';

export interface NotificationContentParams {
    session: Session;
    type: 'permission' | 'input' | 'completion' | 'error';
    permissionName?: string;
    permissionReason?: string;
    customMessage?: string;
}

/**
 * 生成具体的通知内容（包括优化的标题）
 */
export function generateNotificationContent(
    params: NotificationContentParams
): { title: string; body: string } {
    const sessionName = getSessionDisplayName(params.session);

    // 截断会话名称（标题用最多20字符）
    const truncatedSessionName = sessionName.length > 20
        ? sessionName.slice(0, 20) + '...'
        : sessionName;

    switch (params.type) {
        case 'permission':
            return {
                // 标题：包含会话名称 + 状态
                title: t('notifications.permissionTitle', {
                    session: truncatedSessionName
                }),
                // 内容：具体的权限和原因
                body: t('notifications.permissionBody', {
                    permission: params.permissionName || t('notifications.unknownPermission'),
                    reason: params.permissionReason || ''
                })
            };

        case 'input':
            return {
                // 标题："会话名称" 等待指令
                title: t('notifications.inputTitle', {
                    session: truncatedSessionName
                }),
                // 内容：具体在等什么
                body: params.customMessage || t('notifications.waitingForCommand')
            };

        case 'completion':
            return {
                // 标题："会话名称" 已完成
                title: t('notifications.completionTitle', {
                    session: truncatedSessionName
                }),
                // 内容：完成了什么
                body: params.customMessage || t('notifications.taskCompletedDefault')
            };

        case 'error':
            return {
                // 标题："会话名称" 遇到错误
                title: t('notifications.errorTitle', {
                    session: truncatedSessionName
                }),
                // 内容：具体错误信息
                body: params.customMessage || t('notifications.unknownError')
            };
    }
}
```

### Phase 5: 优化会话名称显示

```typescript
/**
 * 获取会话的显示名称
 */
export function getSessionDisplayName(session: Session): string {
    // 优先使用用户设置的名称
    if (session.name) {
        return session.name;
    }

    // 如果有最近的用户消息，使用前30个字符
    const recentUserMessage = session.messages
        ?.filter(m => m.type === 'user')
        ?.slice(-1)[0];

    if (recentUserMessage?.text) {
        const preview = recentUserMessage.text.slice(0, 30);
        return preview.length < recentUserMessage.text.length
            ? `${preview}...`
            : preview;
    }

    // 最后使用会话ID的前8位
    return `Session ${session.id.slice(0, 8)}`;
}
```

## 预期效果

### 用户体验改进

1. **清晰度提升**：用户一眼就能看到是哪个会话需要注意
2. **可操作性**：明确知道需要做什么（授权 vs 输入指令）
3. **多会话管理**：同时运行多个会话时不会混淆

### 技术收益

1. **可维护性**：统一的通知内容生成逻辑
2. **可扩展性**：容易添加新的通知类型
3. **国际化**：完整的多语言支持

## 注意事项

1. **性能**：通知生成不应该影响主流程性能
2. **隐私**：确保通知内容不泄露敏感信息
3. **长度限制**：考虑各平台通知长度限制
4. **向后兼容**：确保旧会话也能正常显示通知
